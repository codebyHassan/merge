Deliverable · What makes it stand out

Six things typical HR dashboards don't do

Each is buildable now from existing data and reads as premium.

① One approvals inbox, not four

Leave, requisitions, offers and document sign-offs land in a single queue with aging chips (green → amber → red by days waiting).

Typical tool: approvals scattered per module

② A digest that talks

A one-sentence, login-time summary of what needs the user today — assembled from live counts, not a static banner.

Typical tool: static welcome text

③ Availability, not just attendance

Managers see who's actually reachable now — in, late, on leave, absent — by fusing punches with approved leave over the org tree.

Typical tool: raw punch list

④ Proactive expiry radar

Documents nearing expiry surface weeks ahead with urgency bars — compliance before it becomes a problem.

Typical tool: find out when it lapses

⑤ Culture built-in

Anniversaries and new joiners create belonging without needing a separate Engagement module licensed.

Typical tool: paywalled module

⑥ Aging & SLA awareness

Every pending item shows how long it's waited (AssignedDate → now), nudging faster action and exposing bottlenecks.

Typical tool: no time context
Deliverable · Build approach

Bootstrap layout & low-risk delivery

A single DashboardViewModel assembled in DashboardController.Index(); each widget a Razor partial gated by HasControllerAccess(). Charts via a tiny inline SVG/Chart.js — no heavy framework.
