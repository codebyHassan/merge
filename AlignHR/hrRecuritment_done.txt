ROOT CAUSE FIXED
  - Original design routed approvals via HR_WorkflowStep.RoleId
  - All HRBPs saw ALL HRBP approvals; no department-level ownership
  - Incorrect approval visibility and poor scalability for enterprise orgs
  - Replaced with employee-level assignment resolved per department

==========================================================
  SPRINT 6 — BUSINESS WORKFLOW IMPLEMENTED
==========================================================

  Application (Accepted candidate)
          ↓
  Create Offer  [HrOffersController.Create]
          ↓
  Offer saved as Draft  (Version 1 auto-created)
          ↓
  Optional: Revise Salary  [AddVersion] — versions tracked, never overwritten
          ↓
  Submit for Approval  [SubmitForApproval]
          ↓  Status = PendingApproval
  Add Approver(s) at any level  [AddApprover]
          ↓
  Each Approver takes action  [ApproveOffer → Approve / Reject]
          ↓  All approved → Status = Approved
          ↓  Any rejected → Status = Rejected
  Send Offer to Candidate  [Send]
          ↓  Status = Sent
  Record Candidate Response  [RecordResponse]
      • Accepted         → Status = Accepted
      • Rejected         → Status = Rejected
      • NegotiationReq.  → Status unchanged (negotiate then revise)
      • Expired          → Status = Expired
          ↓
  Record Hiring Decision  [HiringDecision]
      • Hired   → candidate stage set to "Selected", requisition closed
      • Hold    → no downstream change
      • Rejected → no downstream change
          ↓
  Complete ATS — candidate is hired


BUSINESS RULES ENFORCED
  - Offer can only be submitted from Draft status
  - Only Approved offers can be sent
  - Salary revisions are forbidden once Accepted or Hired
  - One offer per application (enforced in service + unique index)
  - Offer Number auto-generated: OFF-YYYY-NNNN (no gaps, no duplicates)
  - Requisition stays open until Hired (not on offer creation or sending)
  - Document files stored as paths only (no binary in SQL)
  - Approval levels are flexible (not hardcoded), added by recruiter at runtime
